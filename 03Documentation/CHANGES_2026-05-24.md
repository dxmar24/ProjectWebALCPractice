# ALCSystem v2 - Changes - 2026-05-24

## Product Direction

American Latin Class was redesigned as a complete dance academy web system with
three public areas and a role-based school portal.

## Public Website

- Rebuilt the home page as a presentation page for new visitors.
- Added a dedicated pricing page with monthly styles starting at `$35`.
- Added three promotional offer cards that redirect to the enrollment request.
- Updated the enrollment flow so public visitors submit a request and directors
  can contact them outside the system.
- Removed public student self-attendance from the main navigation.
- Repurposed `attendance-kiosk.html` as a school-computer teacher check-in
  station.

## Design System

- Added Bootstrap 5 and Bootstrap Icons through CDN.
- Reworked the visual language around dark graphite gray and duck yellow.
- Kept cards, tables, forms, and navigation compact and modular for usability.
- Rebuilt the dashboard layout with a persistent role sidebar and focused
  module views.

## Student Portal

- Added student progress overview.
- Added schedule cards by level.
- Added upcoming academy events.
- Added monthly attendance metrics and a calendar visualization.
- Kept attendance read-only for students.

## Teacher Portal

- Added teacher work summary with worked hours, late records, and estimated pay.
- Added student attendance control from the teacher portal.
- Added class planning submission with optional planning document URL.
- Added teacher work log based on teacher check-in records.

## Director Portal

- Added operational overview by branch.
- Added student management for creating, deactivating, and updating scholarship
  percentage/status.
- Added teacher management for creating and deactivating teachers.
- Added teacher payroll visibility based on attendance records at `$12` per
  class hour.
- Added planning review table for submitted teacher class plans.
- Preserved finance and B2 professional event modules.

## Backend/API

- Added `25%` as a valid scholarship percentage.
- Added teacher attendance check-in endpoint:
  `POST /api/teacher-attendance/check-in`.
- Added teacher payroll calculation service.
- Added attendance listing endpoint:
  `GET /api/attendance-records`.
- Added teacher management endpoints:
  `GET/POST/PATCH/DELETE /api/teachers`.
- Added student management endpoints:
  `POST/PATCH/DELETE /api/students`.
- Added class plan listing endpoint:
  `GET /api/class-plans`.
- Added optional `document_url` support to class plans.
- Added teacher attendance fields to `attendance_records`:
  `expected_start_time`, `duration_hours`, and `pay_rate`.

## Validation And Routing Hardening

- Added dashboard deep links through Netlify rewrites:
  `/dashboard/overview`, `/dashboard/students`, `/dashboard/teachers`,
  `/dashboard/payroll`, `/dashboard/planning`, `/dashboard/finance`, and
  `/dashboard/events`.
- Added safer API response parsing so HTML deployment errors are reported as
  backend/deploy issues instead of raw `Unexpected token '<'` JSON errors.
- Added password visibility toggles for login and director-created teacher
  passwords.
- Cleared login credentials on browser history restore and redirected protected
  dashboard pages back to login after sign out.
- Reinforced frontend validations for Ecuadorian national ID, email, phone,
  names, URLs, numbers, date/month fields, and controlled option values.
- Reinforced backend validators for director-managed students, teacher accounts,
  attendance records, class plans, and public enrollment.
